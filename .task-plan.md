# Implementation Plan: t5 — Web 交互日志 Tab

## Overview

在 TaskDetail 页面新增"交互日志"Tab，按时间顺序渲染每轮交互（user / assistant），包括文本消息、tool 调用（可折叠）、token 统计等。整体视觉风格类似 Claude Code 终端的交互流，而非聊天气泡。

### 依赖关系
- t4 已完成 `getTaskInteractions` handler（`discord/api/routes/tasks.ts:410-452`），但**路由未注册**到 `server.ts`
- 数据来源：`interaction_log` 表，通过 `InteractionLogRepository.findBySession()` 查询

**设计要点**（终端日志风格，非聊天气泡）：

- **Assistant entries**：
  - 浅灰/白背景

### Step 5: InteractionLog 组件

**新建文件**: `web/src/components/tasks/InteractionLog.tsx`

**Props**:
```ts
interface InteractionLogProps {
  threadId: string;
}
```

**职责**：
1. 调用 `useTaskInteractions(threadId)` 获取数据
2. 将 `interactions` 按 `turn_index` 分组（`Map<number, InteractionLogEntry[]>`）
3. 按 turn_index 升序渲染 `InteractionTurn` 列表
4. 处理 loading/empty/error 状态

**分组逻辑**：
```ts
const turnMap = new Map<number, InteractionLogEntry[]>();
for (const entry of interactions) {
  const existing = turnMap.get(entry.turn_index) ?? [];
  existing.push(entry);
  turnMap.set(entry.turn_index, existing);
}
const sortedTurns = [...turnMap.entries()].sort((a, b) => a[0] - b[0]);
```

**统计摘要栏**（顶部，用 Ant Design `Space` + `Tag`）：
- 共 N 轮交互
- 总 input tokens / output tokens
- 总 cost（如有）

**Empty state**：
- `session_id === null`：`<Empty description="此任务没有 Claude 会话" />`
- `interactions.length === 0`：`<Empty description="暂无交互记录" />`

**Loading state**：
- 使用 `<Spin>` 居中显示（与 TaskDetail 现有 loading 风格一致）

## Key Decisions

1. **终端日志风格 vs 聊天气泡风格**
   - 选择终端日志风格（纵向时间线，左侧色条区分角色），更贴近 Claude Code 终端体验
   - 不复用 `MessageHistory` 的气泡样式 — 交互日志数据结构不同，需要支持 tool_use 折叠和 token 统计

2. **数据分组策略**
   - 按 `turn_index` 分组，每组可能包含多条 entries（一轮中 assistant 可能有 text + tool_use + tool_result）
   - 组内按 role 排序（user 在前），assistant 内按 content_type 显示

3. **交互日志 Tab 默认选中**
   - 交互日志是此 feature 的核心产出，设为默认 Tab
   - 消息历史（Discord 消息）作为辅助 Tab 保留

4. **刷新频率 10s**
   - 交互日志变化频率低于任务状态（5s），10s 刷新足够
   - 任务执行中日志会持续增长，需自动刷新

5. **可折叠 Tool 调用**
   - 默认收起 tool_use，只显示工具名 — 减少视觉噪音
   - 展开后显示 summary_text 全文
   - 使用 Ant Design `Collapse` 组件，`ghost` 样式保持轻量

6. **路由注册补漏**
   - t4 实现了 handler 但未注册路由到 server.ts，本任务 Step 1 修复

## Edge Cases / Risks

1. **大量交互记录的性能**
   - 长时间任务可能有 100+ 轮交互
   - 缓解：初版不做虚拟滚动，原生渲染。Collapse 组件 `destroyInactivePanel` 减少 DOM 节点
   - 如实际出现性能问题，后续可加分页或虚拟滚动

2. **session_id 为 null**
   - 任务可能没有 Claude 会话（刚创建或非 code 任务）
   - API 已处理（返回空 interactions），前端显示 Empty 组件

3. **content_type 值未知**
   - 已知类型：`text`, `tool_use`, `tool_result`, 可能还有 `text+tool_use`
   - 未知 content_type 按普通文本渲染（fallback）

4. **summary_text 为 null**
   - 部分 entry 可能没有摘要
   - 显示占位文本如 `(无内容)` 或根据 content_type 推断默认文本（如 tool_use 显示 `(工具调用)`）

5. **API 404 — 路由未注册**
   - 这是 t4 遗留问题，Step 1 必须先修复
   - 前端 hook 的 error 状态会正确处理 API 错误

6. **Tabs 切换不丢失数据**
   - Ant Design Tabs 默认不销毁非活跃 Tab 的 DOM（`destroyInactiveTabPane` 默认 false）
   - SWR 缓存确保切换回来时不重新 loading
